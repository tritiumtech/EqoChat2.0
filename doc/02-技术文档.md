## 四、记忆系统技术架构

### 4.1 三层记忆存储

| 层级 | 位置 | 触发条件 | 功能 |
|------|------|----------|------|
| **热记忆** | 内存 | 当前对话活跃使用 | 即时上下文、今日频繁访问事实 |
| **温记忆** | 本周MD + MEMORY.md | 心跳整理后保留 | 近期重要事实、高频行为模式 |
| **冷记忆** | 云端图数据库 | Recency < 0.3 | 低频归档事实、历史关系网络 |

### 4.2 Recency评分算法

```typescript
class RecencyScorer {
  calculate(memoryItem: MemoryItem): number {
    const daysSinceAccess = (Date.now() - memoryItem.lastAccess) / (1000 * 86400);
    const timeDecay = Math.pow(0.95, daysSinceAccess);
    const accessBoost = Math.min(1.0, memoryItem.accessCount / 10);
    const importance = memoryItem.importanceScore;
    
    return Math.min(1.0, 0.4 * timeDecay + 0.3 * accessBoost + 0.2 * importance);
  }
}
```

### 4.3 冷记忆图数据库模型

```cypher

  recency: float,
  last_access: datetime

  confidence: float
```

### 7.2 关键设计原则

1. **开放接入**: 支持任意符合W3C标准的DID
2. **主人责任制**: 智能体必须绑定真实人类账户
3. **双轨信用**: 犯罪记录（跨平台）+ 社会信用（平台内）
4. **本地优先**: 敏感数据本地处理，非敏感可上云
5. **渐进信任**: 基于行为历史动态调整权限等级
