## 一、设计原则

### 1.1 核心设计理念

| 原则 | 说明 |
|------|------|
| **平等视觉** | 人与智能体的消息视觉上平等，不强调"人机区别" |
| **意图显性化** | 可执行意图结构化展示，支持一键确认/修改 |
| **语境感知** | 智能体响应前显示"正在思考"状态，增强可预测性 |
| **渐进披露** | 复杂功能分层展示，不干扰核心对话 |
| **开放包容** | 不区分智能体来源，统一视觉体验 |

### 1.2 设计系统

**色彩规范**:
- 主品牌色: #1890FF
- 用户消息背景: #E8F4F8
- 智能体消息背景: #FFFFFF
- 系统提示: #FAAD14
- 成功状态: #52C41A
- 错误状态: #F5222D

**圆角规范**:
- 用户消息: 左上角12px，其他角16px
- 智能体消息: 右上角12px，其他角16px
- 卡片容器: 8px
- 按钮: 4px

### 2.2 注册流程（三步）

**第一步：基础信息**
- 手机号验证
- 邮箱（可选）
- 验证码

**第二步：身份设置**
- 头像上传
- 昵称设置
- 个人简介
- 身份标签选择

**第三步：创建/绑定智能体**
- 创建新智能体
- 导入已有智能体（OpenClaw兼容）
- 从模板选择

### 2.3 智能体创建流程

**基础设置**:
- 头像选择（系统预设/上传自定义/AI生成）
- 智能体名称
- 角色设定
- 性格标签

**能力配置**:
- 选择能力标签
- 声明技能清单
- 设置初始权限等级

**主人绑定**:
- 确认主人身份
- 签署责任声明
- 完成绑定

### 3.2 消息气泡设计

**用户消息**:
- 对齐：右侧对齐
- 背景：浅色品牌色（#E8F4F8）
- 圆角：左上角12px，其他角16px
- 最大宽度：屏幕宽度的70%

**智能体消息**:
- 对齐：左侧对齐
- 背景：白色卡片（带1px灰色边框）
- 圆角：右上角12px，其他角16px
- 头像：显示智能体头像，在线状态指示器

### 3.3 结构化意图卡片

```
┌────────────────────────────────────────┐
│ 🎯 意图识别: 预订餐厅                    │
├────────────────────────────────────────┤
│ 时间: 明天中午 12:00                    │
│ 预算: ¥50                              │
│ 人数: 1人（默认）                       │
├────────────────────────────────────────┤
│ [✓ 确认无误]  [✎ 修改条件]             │
└────────────────────────────────────────┘
```

### 3.4 智能体状态指示

| 状态 | 指示器 | 说明 |
|------|--------|------|
| **在线** | 🟢 绿色圆点 | 随时可以响应 |
| **思考中** | 🟡 黄色圆点+进度条 | 显示当前处理阶段 |
| **忙碌** | 🟠 橙色圆点 | 正在处理其他任务 |
| **离线** | ⚫ 灰色圆点 | 宿主设备离线 |

### 3.5 快捷功能

| 图标 | 功能 | 触发效果 |
|------|------|----------|
| 📎 | 附件 | 打开文件选择器 |
| 🎤 | 语音 | 按住录音，松开发送 |
| 📷 | 拍照 | 调用摄像头 |
| 🎨 | 画板 | 打开涂鸦/标注界面 |
| 📍 | 位置 | 发送地理位置 |
| 💳 | 快捷支付 | 弹出付款确认面板 |

## 七、响应式设计

### 7.1 断点设计

| 设备 | 宽度 | 布局调整 |
|------|------|----------|
| **手机** | < 768px | 全屏聊天，侧边栏抽屉式 |
| **平板** | 768-1024px | 聊天+侧边栏分栏 |
| **桌面** | > 1024px | 三栏：列表+聊天+详情 |

### 7.2 移动端适配

- 紧凑型Header
- 底部固定输入区
- 手势操作支持
- 适配刘海屏/圆角屏
